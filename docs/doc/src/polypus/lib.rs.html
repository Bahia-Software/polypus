3//! # Polypus Version 0.2.0
4//!
5//! Polypus is a distributed quantum computing library designed to optimize the execution of quantum algorithms by distributing computation 
6//! across available hardware resources.
7//! 
8//! Polypus is desgined to be agnostic to the underlying hardware infrastructure. Currently, it support two infraestructures:
9//! - **local**: Runs on local infraestructure (e.g. a local server).
10//! - **qmio**: Platform to simulate distributed quantum computing using multiple emulated QPUs. This infraestructure is desgined by CESGA (<https://github.com/CESGA-Quantum-Spain/cunqa>).
11//!
12//!
13//! ## Features
14//! - Run a quantum circuit.
15//! - Run a quantum circuit using multiple QPUs.
16//! - Train a QAOA using differential evolution.
17//!
18//! ## Examples
19//! Polypus is directly imported from Python. 
20//! ```python
21//! import polypus
22//! ```
23//!
24//! To run a quantum circuit, you can use the `run_quantum_circuit` function:
25//! ```python
26//! result = polypus.run_quantum_circuit(qft_circuit, shots=1000, infraestructure="local")
27//! ```
28//!
29//! We can distribute the quantum circuit shots using multiple QPUs:
30//! ```python
31//! result = polypus.run_quantum_circuit(qft_circuit, shots=1000, infraestructure="local", n_qpus=10)
32//! ```
33//! We can directly import an optimization algorithm. For example, if we define a QAOA circuit and specify how to compute its expectation value, 
34//! we can optimize this circuit using differential evolution. Polypus takes care of optimizing the execution by distributing the shots across 
35//! the available hardware resources.
36//! ```python
37//!polypus.differential_evolution(
38//!   qc=qc, 
39//!   shots=N_SHOTS, 
40//!   n_qpus = N_QPUS, 
41//!   expectation_function=bitstring_to_obj, 
42//!   generations=MAX_GENERATIONS, 
43//!   population_size=POPULATION_SIZE, 
44//!   dimensions=2*layers, 
45//!   infraestructure="qmio", 
46//!   id=id,
47//!   tolerance=TOL,
48//!   nodes=N)
49//! ```
